Authenticated pages
🔒 Displaying pages that require login ▼ Expand
The display board cannot log into websites on your behalf. Browsers block cross-origin access to iframe content for security reasons, so there is no way to inject credentials or automate login flows. Below are the practical options, roughly in order of recommendation.
★ Best Use an embed or public link
Most dashboard tools provide a shareable or embed URL that bypasses login entirely. Check your tool before doing anything else.
Grafana → Share → Embed Power BI → Publish to web Datadog → Shared dashboards Statuspage → already handled Kibana → Short URLs Domo → Embed card
✓ Recommended Dedicated browser session on the display PC
Create a service account (e.g. display@company.com) with read-only access to the dashboards you want to show. Sign into that account in a separate Chrome or Edge profile on the display PC, tick Stay signed in, and open the Display Board in that profile. The browser already holds the session cookies, so iframes load without prompting for login. Set the profile to not clear cookies on close.
1Create a read-only service account in your directory (Entra ID, Google Workspace, etc.)
2Open a new Chrome/Edge profile on the display PC and sign in as that account
3Browse to each dashboard URL and confirm it loads without re-prompting
4Open the Display Board in that same profile and add those URLs as slides
5In browser settings, disable “Clear cookies on close” for that profile
⚠️ Sessions still expire eventually. The URL reload interval setting (per slide) can help — refreshing the iframe will re-use the existing session cookie if it is still valid.
ℹ️ Fallback Keep a browser tab open alongside the board
For pages that re-authenticate frequently (e.g. MFA every 24h), the simplest fallback is to keep the page open in a regular browser tab on the same machine. The tab maintains its own session independently. The Display Board can run in fullscreen on a second monitor or second window, with the authenticated tab available if you need to refresh or re-auth.
✗ Not possible Automatic login via stored credentials
The board cannot fill in usernames and passwords, click login buttons, or handle MFA flows inside iframes. This is enforced by the browser’s same-origin security policy and cannot be worked around in a client-side tool. Any site that shows a login screen inside the iframe will stay on that screen.
💡 Tip: Add a session note to any URL slide as a reminder of which account to use. Notes are only visible on this config page — never shown on the board itself.
